# Private beta plan

## Staged rollout

1. **Supervised alpha: 3 users** — install help, first-session walkthrough, structured feedback.
2. **Private beta: 10 users** — broader usage; triage loop from `TRIAGE.md` applies.
3. **Expand to 20 only if** P0/P1 issue rate in the 10-user cohort is below the threshold agreed before expanding (see release checklist and audit plan).

## Target pool (10–20 long-term)

Recruit from friends, university peers, and small dev communities who accept alpha risk.

## Onboarding script

1. Install from the latest release.
2. Configure provider key in Settings > AI.
3. Run first commands: `>app`, `>find`, `>remember`, `>summarize`.

## Structured feedback template

- Environment: Windows version, RAM, WebView2 version.
- Daily-use frequency: 0/1-5/6-20/20+ commands per day.
- Failures: exact command + observed result.
- Trust issues: confirmations, undo, audit clarity.
- Feature requests: top 3.

## Triage SLA

- Classify each report within **72h** (see `TRIAGE.md`).
