# Solo maintainer sustainability

This project is built by a **student / zero-capital** maintainer. Burnout and scope creep are project risks, not just personal ones.

## Weekly rhythm (aspirational)

- **Saturday:** maintenance block — triage, small fixes, release hygiene, docs truth pass.
- **Sunday:** **no-code by default** (rest, or non-code: issues, user replies, learning).

Skip this when deadlines demand it, but **do not** let “every day is a crunch day” become the default for months.

## Scope

- **`Blueprint.md`** is the **scope authority**; “cool ideas” go to a **GitHub issue** or `docs/FUTURE.md`, not into `main` on impulse.

## Support load

- Use **GitHub Issues** for bug reports and feature discussion; there is no paid SLA.
